# Feasibility report — anima-ai

_Compiled 2026-07-13 from five research passes (opposition, user sentiment, market landscape, market gaps, build tooling). Provenance caveat: the research runs were stopped mid-pipeline before their adversarial-verification stage ran, so claims below are single-sourced unless noted. The single most verdict-critical claim (HyperFrames) was independently re-verified by direct fetch on 2026-07-13. Star counts and install numbers should be treated as approximate._

## Verdict

Build it — but on a corrected premise. The original pitch's implied claim, "nobody owns agent-authored video," is false: Remotion ships official Agent Skills for Claude Code/Codex/Cursor (the most-installed video skill, ~126k+ installs), HeyGen's HyperFrames (Apache-2.0, ~34.7k stars within months, verified 2026-07-13) is explicitly "built for agents" with deterministic rendering and golden-baseline regression tests, Revideo/Midrender is YC-backed in the same thesis, and prompt-to-explainer SaaS (Golpo $4M/120k users, Simi, Knowlify) is a funded category. LLM-to-Manim pipelines are commoditized on GitHub and actively benchmarked in academia (TheoremExplainAgent, Code2Video/ICML 2026, SimuScene). What is genuinely unclaimed — and the only ground worth building on — is the specific intersection anima-ai scoped in the interview: **deterministic *semantic* verification** (scene-graph assertions + layout/pacing constraints; every competitor verifies via render-success, pixel-drift against goldens, or VLM-watching, and VLM-watching measurably barely works), **map/history primitives as code** (zero libraries ship them; demand proven by two SaaS map-animation studios and the manual After Effects + GEOlayers incumbent workflow), and **an LLM-first Python API** (the Python slot holds only Manim, whose own maintainers acknowledge an unstable, confusing API that LLMs demonstrably fail against). The two honest risks: the verification-as-product bet — encoding taste as deterministic constraints at 15–20-minute scale — is unproven by anyone, and layout quality is precisely the failure mode all research identifies as unsolved; and the window is real but not lazy — HyperFrames or Remotion could bolt on semantic verification, so differentiation must be structural (scene-graph-native, Python, maps, paint styles) rather than a feature they can copy in a quarter. Proceed.

## Opposition research

**Remotion** — the category incumbent. React/TypeScript, renders via headless Chromium; ~53k GitHub stars, ~60k weekly npm downloads, 4M+ installs/month claimed. Business Source License with paid company tiers ($0/$50/$200 by ARR) — a repeatedly documented adoption blocker (HN threads of developers rejecting it on licensing grounds; a whole `remotion-alternative` GitHub topic exists because of it). Bootstrapped-adjacent (~$270k raised, ~200 paying customers, profitable 2024). Shipped official Agent Skills Jan 2026 (~30 rule files for Claude Code/Codex/Cursor; reportedly 126k–155k installs) — the strongest direct mover on the agent-authored-video wedge. Critical gap, from its own docs: the testing story is DOM-level React component testing; no frame comparison, no scene-graph assertions, no machine verification of rendered output. Agents using Remotion still iterate by rendering and eyeballing.

**HyperFrames (HeyGen)** — verified by direct fetch 2026-07-13. Apache-2.0, TypeScript/Node, "Write HTML. Render video. Built for agents." ~34.7k stars, 3.3k forks. Deterministic rendering ("same input, same frames, same output — built for CI"), golden-baseline regression harness (~240MB of reference MP4s in Git LFS), 20+ agent skills. This kills any claim that "deterministic rendering for agents" is a moat by itself. What it does not have: semantic verification (no scene-graph assertions, no layout checks — confirmed absent), Python authoring, animation-authoring API (it's an HTML-to-video rendering layer), maps, or any stylized-explainer aesthetic.

**Revideo / Midrender** — MIT fork of Motion Canvas, YC S23, positioned for automated pipelines, monetized via hosted rendering. Founder pivoted to Midrender (AI motion-graphics product); Revideo OSS is now likely under-maintained (~3k weekly downloads). Signal: VCs funded the "video as code for automation" thesis, and the pure-library play alone didn't sustain a company.

**Motion Canvas** — MIT, TypeScript, explainer-focused; ~16k stars but original line no longer actively maintained (community fork "Canvas Commons" carries it). Browser-only rendering, no headless path — structurally weak for agent automation.

**Manim / ManimCE** — the only Python incumbent. Powerful output, but: notoriously painful install (ffmpeg, Cairo, LaTeX), maintainer-acknowledged public/internal API confusion (issue #3792, open 2+ years), unstable API, documented render-performance defects, and an API that LLMs verifiably fail against (render-failure rates ~33% for untuned models; TheoremExplainAgent measured Claude 3.5 Sonnet at 2.1% success without scaffolding). An entire cottage industry of wrappers (generative-manim, manim-generator, Manimator, TheoremExplainAgent, Code2Video) exists specifically to compensate — via retry loops, log parsing, and VLM frame review bolted on outside the library. That scaffolding is the demand signal for an LLM-first animation library.

**AI whiteboard/explainer SaaS** — Golpo (YC S25, $4M, 120k+ users claimed), Simi (YC), Knowlify (YC), plus legacy VideoScribe/Doodly and Animaker's AI bolt-on. All prompt-in/opaque-video-out for non-technical users: no code artifact, no API an agent can author against, no verification (Mapimator literally instructs users to manually review output). They prove demand for the output genre; none compete for the agent/library slot.

**Map animation specifically** — MapAnimation.io and Mapimator are closed GUI SaaS studios marketing history/battle-map use cases. The professional workflow is After Effects + the paid GEOlayers plugin, fully manual. Open data exists (historical-basemaps GeoJSON, Natural Earth); no library turns it into animation primitives. This sub-niche is genuinely empty on the code side.

**Research prior art** — MoVer (ACM TOG) is the closest thing to anima-ai's headline claim: a first-order-logic DSL that deterministically checks spatio-temporal properties of SVG motion graphics and feeds reports back to an LLM. It exists as a research artifact, not a product, and it validates rather than forecloses the approach. SimuScene, SPIRAL, VideoGameQA-Bench all verify rendered output by VLM-watching — and quantify how poorly that works (below).

## User sentiment

Complaints that map directly to anima-ai's openings:

- **Manim**: dependency hell at install ("meticulously configured environment", ffmpeg/LaTeX failures); slow full re-renders for every small change with low-quality-preview coping loops; caching that plays back stale animations (`--disable_caching` folklore); progressive render slowdown in common patterns (maintainer-confirmed Cairo defect); an hour lost hand-tweaking label positions by visual trial-and-error. The iterate-render-watch loop is the pain; "verified without watching" attacks it directly.
- **Remotion**: the license is the top recurring community complaint and demonstrably changes adoption decisions; headless-Chrome rendering cost/slowness; frame-count-driven choreography awkward for long timelines.
- **LLM+Manim products**: users of a 3blue1brown-style AI generator report "words overlapping", videos cutting off mid-explanation, confidently wrong content — the two failure classes (broken layout nobody catches; nobody verifies output) are exactly the verification gap.
- **Whiteboard SaaS (VideoScribe/Doodly)**: billing/upsell hostility, platform lock-in, catastrophic account/data loss, expressiveness ceilings ("can't animate more than one item at a time"), no MP4 export, rough hand-drawn text stroke paths. Local plain-code project files and open source sidestep the entire class.
- **AI video SaaS (Pictory/InVideo/Synthesia)**: irrelevant AI-picked stock visuals, generic output, credit systems that punish iteration (contrast: re-rendering code is free), unverifiable output requiring manual review of every generation.

The bar to clear (recurring praise): VideoScribe/Doodly's ease-of-use and the hand-drawn aesthetic itself; Manim's output quality and programmatic precision; Motion Canvas's documentation quality and live preview; Remotion's fast preview loop and "videos as code" testability. Notable craft detail users notice: Doodly's pen not tracing letter strokes accurately — procedural stroke quality on *text* is an under-served detail.

## Market landscape

- **Generative pixel video (Sora/Veo/Kling/Runway) is not a competitor for this genre.** Single-pass clips top out ~15–20s; characters drift across stitched clips; no cross-shot consistency mechanism; text rendering garbles; per-second API costs ($0.05–$0.75/sec — a 30s Sora 2 clip ≈ $22.50) versus zero-marginal-cost local deterministic rendering. Multi-minute coherent explainers are structurally out of reach for pure generation. Capital is concentrated there ($1.82B into AI video creation in 2025, Runway $860M total) — meaning the code-driven/deterministic lane is comparatively uncontested by funded players.
- **The AI-slop backlash is a tailwind.** Consumer preference for gen-AI creator content collapsed (60%→26%); YouTube's July 2025 "inauthentic content" policy demonetizes mass-produced templated output but explicitly keeps AI-assisted, editorially distinctive content monetizable; peer-reviewed work argues expert-quality AI tooling benefits creators once output clears the quality bar. Distinctive, author-controlled, stylized animation sits on the right side of every one of these lines — but only if output is genuinely distinctive, which loops back to the taste-encoding bet.
- **Agent-skill distribution is the adoption channel.** SKILL.md is now a cross-vendor open standard (Claude Code, Codex CLI, Cursor, Gemini CLI, Copilot); MCP is foundation-governed with 10k+ public servers; Remotion's skill got ~25k installs in week one. A Python library's go-to-market is: pip + a first-class skill + llms.txt/AGENTS.md + version-pinned docs (Context7-style) so agents don't hallucinate the API. Poorly structured agent docs measurably *hurt* (one study: −success, +20% cost), so this surface is engineering, not marketing.
- **Economics of the genre**: studio 2D explainer runs $1,500–$4,000+/min and 4–8 weeks; Kurzgesagt ~1,200 hours/video. The supply of premium explainer animation is scarce because production is brutal — the value of collapsing it to an agent loop is large and quantifiable.
- **All three code-to-video incumbents are TypeScript.** The Python slot — the language LLMs write best and the ecosystem agents live in — contains only Manim. That slot is the opening.

## Market gaps and differentiation

Ranked by defensibility:

1. **Deterministic semantic verification of authored animation — unclaimed by any product.** The field's verification ladder today: (a) render-success/exit-code (RITL, most Manim wrappers), (b) pixel-drift against golden baselines (HyperFrames — catches regressions, cannot judge *new* content), (c) VLM-watching (SimuScene, TheoremExplainBench, manim-generator) — which is quantifiably inadequate: ~12% disagreement with humans, 45.2% accuracy on visual-regression tasks (the best model), F1 0.51 on animation-specific defects, non-deterministic across runs, and practitioners report it "barely works". Nobody ships scene-graph-native assertions (positions, overlaps, timing sums, on-screen guarantees) + constraint rules as a product guarantee. MoVer proves the concept in research; anima-ai productizes it. Supporting evidence for the API half: structured machine-readable errors lifted agent task completion +36.7–40.0pp on Anthropic models in a controlled study — and the layout-correctness failure mode this targets is the single lowest-scoring metric in every LLM-animation benchmark (Element Layout 0.52–0.65).
2. **Map/history-geopolitics primitives.** Empty on the library side; demand proven by SaaS entrants and the manual AE workflow; open data ready (Natural Earth, historical-basemaps). Structural, not cosmetic: GeoJSON ingestion, historical borders, territory fills, invasion arrows, label pops as first-class verified primitives.
3. **LLM-first Python API.** "LLM-friendly API design" is a named 2026 pattern with zero animation-library implementations. Manim's failure here is documented by its own maintainers and by every wrapper built to route around it.
4. **Genuinely permissive license.** Remotion's BUSL created durable, documented demand for a real OSS alternative; MIT/Apache positioning is free differentiation.
5. **Paint-style aesthetic packs.** Weakest moat alone (styles can be copied), but combined with 1–3 it defines the product: the whiteboard-SaaS praise shows the hand-drawn look itself is what users buy.

What is *not* a gap (do not claim it): deterministic rendering (HyperFrames, Remotion, Replit all have it), "agent authors video as code" (Remotion's whole agent push), "AI generates explainer videos" (funded SaaS category), "agent self-review loops" (commoditized on GitHub).

Window risk, stated plainly: HyperFrames iterated to 34.7k stars in months and Remotion ships agent tooling as a first-class roadmap item. Either could add a verification layer. The defense is that scene-graph-semantic verification is architecturally native to a purpose-built engine and retrofit-hostile to HTML-screenshot pipelines (their scene state lives in the browser, not in an inspectable graph) — but the paint-style + maps + Python wedge should ship before the giants notice the verification story.

## Recommended build tooling

Renderer:
- **skia-python** as the 2D backend: actively maintained (v144.x, Mar 2026), wheels for all mainstream platforms, ~34× faster than Pillow for vector work, full path/shader API needed for wobbly-stroke/paint-texture rendering. BSD-3. Caveats: pin the exact version (rendering shifts between Skia milestones invalidate goldens); Linux wheels need system GL/EGL libs in CI images; vendored freetype pins text rasterization (good for determinism) but couples security updates to golden regeneration.
- **pycairo** is the fallback (stable, but system-cairo linkage on Linux/macOS is a cross-platform determinism hazard, and no shader/filter surface for paint textures). **Vello**: watch, don't build on — alpha, no Python bindings, GPU-compute is hostile to bit-exact determinism.
- **Architecture lesson from resvg**: bit-exact cross-platform determinism is achievable only by owning the whole stack — bundle fonts, avoid system shaping/raster libraries. Matplotlib never achieved bit-exact text in 20 years and engineered around it with tolerances and text removal. Design decision: bundle fonts, pin freetype via skia-python, and use tolerance-based comparison (pixelmatch-style with anti-aliasing detection) rather than chasing bit-exactness on developer machines; generate CI goldens inside a pinned container (Playwright's concession is the template).

Encoding:
- **PyAV**, not ffmpeg-python (unmaintained since ~2019) and not subprocess pipes. Precedent: Manim v0.19 switched to PyAV (PR #3501) so `pip install` bundles ffmpeg with no system dependency — exactly the friction-free install story anima-ai needs. Cap/pin the PyAV version (Manim had to, PR #4037). Performance patterns with receipts: frame-writing on a separate thread from rendering (Manim PR #3888); single-encoder-process architecture, never per-worker ffmpeg pipes (moviepy #1273 deadlocks, never fixed); frame-to-bytes conversion is the real pipeline bottleneck to optimize.

Maps:
- **shapely + pyproj + topojson** — explicitly not geopandas (pulls pandas + GDAL/pyogrio dead weight and a documented install-reliability risk; GeoJSON parses with stdlib json + `shapely.geometry.shape()`). topojson for topology-aware simplification (naive per-polygon simplify gaps/overlaps adjacent country borders — the exact failure a map engine can't have); its quantized integer coordinates also aid determinism. Pin PROJ version (grid-shift files affect reproducibility). Vendor Natural Earth as pre-converted GeoJSON (martynafford/natural-earth-geojson), version-pinned; historical borders from aourednik/historical-basemaps.

Testing:
- Golden-image workflow per **pytest-mpl**'s pattern (RMS tolerance + pinned rendering stack + per-environment baselines) but framework-agnostic — **pytest-image-snapshot**/pixelmatch-style comparison with anti-aliased-pixel exclusion as the engine (small-maintainer caveat: consider vendoring the comparison logic, it's small). Syrupy exists as a Claude Code skill for pytest snapshot workflows.

Agent-facing CLI (design patterns with evidence):
- Typed exit codes encoding retryability (0 success / distinguish transient from deterministic failure); `--json` structured output on every command (gh CLI's adoption lesson); strict stdout-data/stderr-errors contract; non-interactive defaults; `--help` as a primary agent documentation surface; compact output mode (measured ~40% token savings in a comparable tool); a dry-run/plan mode for render commands.

Distribution/dev tooling that exists today:
- Remotion's official agent skill + wilwaldon/Claude-Code-Video-Toolkit (skills/MCP/commands for Remotion, Manim, FFmpeg pipelines) — study as packaging references. A Manim MCP server exists (write scene → server renders → MP4 back) as a render-loop precedent. digitalsamba's FFmpeg skill covers encoding presets. Ship with the library: SKILL.md (cross-vendor standard), llms.txt, AGENTS.md, version-pinned docs exposed Context7-style.
